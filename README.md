## API для списка активных программ

Это Django‑проект `pk_server` с REST‑эндпоинтом, который возвращает список активных программ повышения квалификации из базы данных.

### Требования

- Python 3.11+ (рекомендуется та же версия, что используется в проекте)
- Виртуальное окружение (`venv`)

### Установка и запуск (Windows)

1. Клонируйте/скачайте проект и перейдите в каталог:

```bash
cd c:\projects\pk_server
```

2. Создайте и активируйте виртуальное окружение:

```bash
python -m venv venv
venv\Scripts\activate
```

3. Установите зависимости:

```bash
pip install -r requirements.txt
```

4. Создайте файл `.env` на основе `.env.example` и задайте значения:

- `SECRET_KEY` — секретный ключ Django
- `DEBUG` — `True` для разработки, `False` для продакшена
- `ALLOWED_HOSTS` — список хостов, с которых будет доступен сервер
- Параметры БД при необходимости

5. Примените миграции и запустите сервер:

```bash
python manage.py migrate
python manage.py runserver
```

6. Вызовите эндпоинт списка активных программ:

```bash
curl http://127.0.0.1:8000/api/active-programs/
```

### Документация API (Swagger/OpenAPI)

Проект включает интерактивную документацию API на основе Swagger/OpenAPI:

- **Swagger UI**: `http://127.0.0.1:8000/api/swagger/` — интерактивный интерфейс для тестирования API
- **ReDoc**: `http://127.0.0.1:8000/api/redoc/` — альтернативный интерфейс документации
- **OpenAPI Schema**: `http://127.0.0.1:8000/api/schema/` — JSON-схема OpenAPI

В Swagger UI можно:
- Просмотреть все доступные эндпоинты
- Увидеть структуру запросов и ответов
- Протестировать API прямо в браузере

### Замечания по безопасности

- Эндпоинт `/api/active-programs/` **не требует аутентификации** и возвращает публичную информацию об активных программах.
- В продакшене рекомендуется:
  - ограничить доступ по IP (firewall, reverse‑proxy) при необходимости,
  - добавить базовую авторизацию или иную защиту на уровне прокси, если требуется ограничить доступ.

### Ручная проверка

1. Убедитесь, что сервер запущен: `python manage.py runserver`.
2. Откройте в браузере `http://127.0.0.1:8000/api/active-programs/` или выполните `curl`‑запрос, как показано выше.
3. Проверьте, что:
   - возвращается код ответа `200 OK`;
   - тело ответа — JSON‑массив;
   - в элементах массива присутствуют ключи `id`, `name`, `direction`, `lead`, `status` и другие поля программы;
   - возвращаются только программы со статусом `active`;
   - данные соответствуют программам в базе данных.
